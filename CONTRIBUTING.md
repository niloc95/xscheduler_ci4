# Contributing

Thanks for your interest in contributing.

This repository is maintained on a **personal account** and uses a **PR-only** workflow. Most contributors will contribute via **forks**.

## Workflow
- Fork the repository
- Create a feature branch on your fork
- Open a Pull Request into `main`

## Local setup
- Install PHP dependencies: `composer install`
- Install Node dependencies: `npm install`
- Build assets: `npm run build`

## Quality bar (PR requirements)
- CI must pass (GitHub Actions)
- Keep changes focused and small
- Prefer tests where applicable
- Avoid adding debug logs (`console.log`, `var_dump`, etc.)

## Security issues
Please do **not** open public issues for security vulnerabilities. See [SECURITY.md](SECURITY.md).
